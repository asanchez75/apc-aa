New Features and Improvements

New Reader Management, Alerts/Digest Module and Web Directory Password Protection Management.

In AA v2.6 introduces a Reader Management module. The Reader management uses an ordinary (well, slightly extended) slice to store information about subscribed site visitors. It can be used in several ways, such as letting people subscribe to Alerts (daily/weekly/monthly/instant digests from the latest items added to a certain slice). You can also use the information stored in this slice for password protecting parts of your website. The user information can be retrieved with the use of views. A nice demo has been made by Jason Diceman - visit RM demo at SourceForge
Documentation:

  • Reader Slice
  • Alerts
  • Anonymous Form
  • Public user permissions

Restricting access of a web server directory to only valid users or groups listed in a Reader Management slice is done by mod_auth_mysql Apache module. See Reader Slice for more information. At present, we only support native http type of authentication, PHP session or cookies authentication will come later.

The list of subscribed readers email addresses can also be used as a base for Mailman powered mailing list (but note that people can't subscribe to AA via Mailman subscribtion mechanism yet).

New Links Module

A support for Yahoo-like hierarchical catalogs of links based on the idea of Open directory - various people can contribute and maintain their own branches of the hierarchy, having them included in their own (aa powered) website, while still being part of the global catalog. Features includes sharing links between many administrators/catalogs. Econnect just launched the new "Open Society Links Catalog" kormidlo.cz. This Catalog is maintained through AA by 23 different organizations at the moment. The output from the catalog is displayed not only on kormidlo.cz, but also on users' pages - just like:

  • http://feminismus.cz/kormidlo.shtml
  • http://mkc.cz/odkazy.shtml
  • http://romea.cz/index.php?id=odkazy

Design of the output is controlled through VIEWs - just like most of the content in AA. Independent catalogs are also supported. Unfortunately, at present there is no English documentation for this feature. We have only a Czech user guide for kormidlo.cz catalog. Volunteers to write English documentation please contact honza.malik@ecn.cz.

Many New Parameters for view.php3 and slice.php3

[JD says: parameters in general need many more examples and clearer technical writing to explain better.]

  • new view.php3 parameters
    • added possibility to specify multiple fields in 'd' parameter of view.php3, e.g.:
      (cmd[246]=d-headline.......1,category........-RLIKE-Enviro)
    • added new "sort" param to view.php3 (set[246]=sort-headline.......1)
    • new special "url" parameter used with "x" view.php3 cmd[] - view.php3?cmd[43]=x-43-url - which means, that such view takes item_id from x=... url parameter (obviously used with slice.php3)
      (we use it to create special view for <title> tag to contain the item headline)
    • new set[]=slices-slice_id1-slice_id2-sli... for view.php3 allows for displaying items from more than one slice through one view
    • added special view.php3 alias _#SELECTED, which is "1" for selected item (given by special set[] view.php3 paramater (like: set[34]=selected-43535) - usable for displaying menu generated from slice items (see leftmenu (topics) at  obcan.ecn.cz))
    • AAnoCONDITION could be used also with cmd[]=d (not only with cmd[]=c)
    • new view.php3 command 'o', which behaves exactly like 'x' but item hit-count is not increased


(See: http://apc-aa.sourceforge.net/faq/#219)

  • new slice.php3 parameters
    • added banner parameter to slice.php3
    • special "dview" slice parameter, which defines used view for discussions (just like ivew and fview parameters)
    • added the Reading Password slice setting for protecting sensitive data in a slice (see: http://apc-aa.sourceforge.net/faq/#slice_pwd


(See: http://apc-aa.sourceforge.net/faq/#sliceparam)

View improvements

  • 'Constant view' now done through a standard view, so you can use conditions, sorting and aliases (conds[], sort[], als[]) in your list of constants
    (See: http://apc-aa.sourceforge.net/faq/#213)
  • New keyword " " can be used in "No item message" parameter in view or slice setup, which will display nothing
  • "No item message" now is unaliased - you can use there AA expressions and aliases
  • Added a lightweight type of view for printing a long list of references (urls) to slice items -  use this for generating an entry point for search robots (with normal views you could often reach memory limits)
  • fixed incorrect order of odd/even rows in itemview (Index)

Related Item Improvements

Related items can now be displayed in user defined order, if not overridden by other sorting. User can easily move related items up and down with arrow buttons - as long as this is allowed  in "Field-setting -> Input type". 

Send Discussion Items to email (mailing list)

Discussions under item can be posted to a mail (probably a mailing list).

It is not new, but now you can use an e-mail template for it (just like in Alerts) and use any discussion alias (like _#SUBJECT_) in e-mail's body or subject. Setting is the same as with view, but the template number is prefixed by "t" letter (like: t21:our-l@list.org.cz). Older approach (through discussion view) is deprecated.
(See: http://apc-aa.sourceforge.net/faq/#messageboard)
